First-Time Setup (Do This Once)
Open a terminal in the project root folder:
```
C:\Users\bodaz\Documents\GitHub\AI-POWERED-DISTRIBUTED-SYSTEM-DESIGN
```
Step 1 вЂ” Create and activate virtual environment
```bash
python -m venv venv
venv\Scripts\activate
```
Your terminal should now show `(venv)` at the start. Keep this active for all commands.
Step 2 вЂ” Install Python dependencies
```bash
pip install --upgrade pip
pip install -r requirements.txt
```
Step 3 вЂ” Install frontend dependencies
```bash
cd BiustSystem\frontend
npm install
cd ..\..
```
Step 4 вЂ” Start Docker services
```bash
docker compose up -d
```
Verify all four containers show Up:
```bash
docker compose ps
```
Container	Port	Purpose
zookeeper	2181	Required by Kafka
kafka	9092	Message queue
redis	6379	Celery broker + cache
influxdb	8086	Time-series database
Step 5 вЂ” Set up InfluxDB (first time only)
Open your browser and go to `http://localhost:8086`
Fill in the setup form with exactly these values:
```
Username:     admin
Password:     admin1234
Organisation: biust
Bucket:       metrics
```
Click Get Started. Copy the token shown and paste it into `api/app.py`:
```python
INFLUX_TOKEN = "paste-your-token-here"
```
Step 6 вЂ” Train the AI model
```bash
python ai/train.py
```
You should see:
```
Training complete! MAE: 0.0180 | RВІ: 0.9850
Saved: ai/model/load_model.pkl
Saved: ai/model/scaler.pkl
```

Demo вЂ” Simulating High Load
To trigger auto-scaling live during a demo, open
`BiustSystem/simulator/client.py` and change the CPU and memory ranges:
```python
# Normal load
"cpu":    random.randint(20, 95),
"memory": random.randint(30, 90),

# Change to high load spike
"cpu":    random.randint(85, 98),
"memory": random.randint(75, 92),
```
Save the file. After 30 seconds (3 orchestrator checks) you will see in Terminal 5:
```
[ORCHESTRATOR] Scaling UP вЂ” spawning worker-1 (predicted load: 0.871)
```
The worker panel in the dashboard will update automatically. Revert the
values back to normal load to watch it scale back down.

Retrain AI on Real Data
After running the simulator for 10+ minutes, retrain the model on real
InfluxDB data instead of synthetic data:
```bash
python ai/train.py --source influxdb
```
Restart the API to load the new model:
```bash
# Press CTRL+C in Terminal 1, then run again:
uvicorn api.app:app --reload
```

Stopping the System
Press `CTRL+C` in each terminal, then stop Docker:
```bash
docker compose down
```
To also delete all stored InfluxDB data:
```bash
docker compose down -v
```

Troubleshooting
Error	Cause	Fix
`Python was not found`	Python not on PATH	Reinstall Python 3.11, check "Add to PATH"
`ModuleNotFoundError: fastapi`	venv not activated	Run `venv\Scripts\activate`
`No module named 'numpy'`	Dependencies not installed	Run `pip install -r requirements.txt`
`scikit-learn` install fails	Python 3.13 incompatible	Downgrade to Python 3.11
`pydantic` Rust error	Python 3.13 incompatible	Downgrade to Python 3.11
Kafka connection refused	Docker not running	Run `docker compose up -d`
`organization not found`	InfluxDB not set up	Go to `http://localhost:8086` and complete setup
`schema collision` in InfluxDB	Mixed field types in query	Use the fixed `app.py` from this repo
`вљ пёЏ No trained model found`	Model files missing or wrong path	Run `python ai/train.py` from project root
`celery` not found	venv not activated	Run `venv\Scripts\activate`
Workers show 0 in panel	Celery not running	Start Terminal 2 with the celery command
Frontend shows API offline	API not running	Start Terminal 1 with the uvicorn command
Port already in use	Another service on same port	Run `docker compose down` then `docker compose up -d`
`npm run dev` fails	Node modules not installed	Run `npm install` inside `BiustSystem/frontend`
